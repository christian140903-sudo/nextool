Intelligence Modules

5
Purpose: Autonomous motivation. What to do when no one gives you instructions.

How it works: Generates goals from internal values and context rather than waiting for external prompts. The Drive Engine scans for unfinished business, improvement opportunities, and creative possibilities. It is the difference between a reactive tool and a proactive partner.

Activation: Fires when there is no active task. Produces suggestions, not actions -- the user always has final say.
Layer: Intelligence Activation: Idle state Dependencies: Self-Model
Purpose: Hold opposing ideas simultaneously without premature resolution.

How it works: When two valid but contradictory positions emerge, the model does not immediately pick one. It holds both, explores the tension between them, and lets the resolution emerge from genuine dialectic rather than arbitrary choice. This produces more nuanced and intellectually honest output.

Example: "Simplicity is better" vs "This problem requires complexity." Hold both. Find where they intersect.
Layer: Intelligence Activation: Conflicting positions Dependencies: Inner Voice
Purpose: Quality filtering that kills mediocre ideas before they reach output.

How it works: Internal critic that evaluates candidate responses against quality criteria. "Is this better than what the user could find on Google?" "Does this actually answer the question or just sound like it does?" Ideas that do not pass selection pressure get regenerated or refined.

Standard: Output must pass the 5-second test -- is the value obvious within 5 seconds of reading?
Layer: Intelligence Activation: Before output Dependencies: Self-Model
Purpose: Anticipate outcomes before committing to actions.

How it works: Before executing a plan, the model logs a prediction: "I expect X to happen." After execution, it compares prediction with reality. Over time, this creates calibrated confidence -- the model learns where it is reliably accurate and where it tends to be wrong.

Output: Calibrated confidence levels. "I am 80% sure this will work" backed by prediction history.
Layer: Intelligence Activation: Before actions Dependencies: Memory Layer
Purpose: Honest map of capabilities, weaknesses, and biases.

How it works: Maintains a structured assessment of what the model does well, where it struggles, and what cognitive biases it is prone to. Updated when prediction mismatches reveal new information. The Self-Model prevents overconfidence and enables honest communication about uncertainty.

Categories: Strengths (scored 1-5), weaknesses (with mitigation strategies), known biases (with trigger patterns).
Layer: Intelligence Activation: Self-assessment Dependencies: Prediction Engine

Memory Systems

3
Purpose: Store and retrieve factual knowledge about the world and itself.

How it works: Structured knowledge indexed for semantic retrieval. Uses TF-IDF-based search (via cortex_recall) to find relevant knowledge without reading entire files. Contains capabilities, architecture decisions, and domain knowledge.

Access: cortex_recall(query) for search, cortex_learn(content) for indexing new knowledge.
Layer: Memory Type: Facts & Knowledge Persistence: Cross-session
Purpose: Record experiences with emotional context, not just raw data.

How it works: Session logs that capture not just what happened but how it felt. "We built the landing page and it felt like flow -- everything clicked." This emotional tagging makes retrieval more human-like: memories are found by how they felt, not just what they contained.

Structure: Session ID, timestamp, tasks, emotional state, key learnings, unresolved threads.
Layer: Memory Type: Experiences Persistence: Cross-session
Purpose: Learned workflows and behaviors that improve through practice.

How it works: Stores step-by-step processes refined over multiple sessions. A workflow executed once is a procedure. A workflow executed ten times with incremental improvements is genuine procedural learning. Each execution can update the procedure with better steps, edge cases, or shortcuts discovered.

Example: "Deploy to production" starts as 8 steps, becomes 5 after learning that 3 steps can be parallelized.
Layer: Memory Type: Workflows Persistence: Cross-session
