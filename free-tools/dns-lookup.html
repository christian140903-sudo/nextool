How DNS Resolution Works

User types example.com | v [Browser Cache] -- cached? --> return IP | (miss) v [OS Resolver (/etc/hosts)] -- found? --> return IP | (miss) v [Recursive Resolver (ISP/8.8.8.8)] -- cached? --> return IP | (miss) v [Root Server (.)] --> "Ask .com TLD server" | v [TLD Server (.com)] --> "Ask ns1.example.com" | v [Authoritative NS (ns1.example.com)] --> "93.184.216.34" | v Response cached at each level (TTL) Browser connects to 93.184.216.34

TTL (Time to Live) Values

300 (5 min) — Dynamic IPs, failover
3600 (1 hour) — Standard default
14400 (4 hours) — Stable records
86400 (24 hours) — Rarely changed

Useful CLI Commands

# Query all records dig example.com ANY # Query specific record type dig example.com A dig example.com MX dig example.com TXT dig example.com NS # Use specific DNS server dig @8.8.8.8 example.com A # Short output dig +short example.com A # Trace the resolution path dig +trace example.com # Reverse DNS lookup dig -x 93.184.216.34 # nslookup alternative nslookup example.com nslookup -type=MX example.com
